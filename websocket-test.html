Проверка WebSocket /ws

Открой этот файл через локальный сервер (см. ниже) или с фронта на том же origin. Токен — поле access_token из POST /api/v1/auth/login.

Статус: не подключено

Примеры: send_message с chat_id и encrypted_content (объект); read_receipt с message_id.

file://: браузер может блокировать WebSocket. Запусти из корня backend: npx --yes serve -p 5500 и открой страницу с порта 5500, а в поле «Базовый URL API» оставь http://localhost:8000 — к API подключается WebSocket.
